# 任务拆分 · tasks.md

> 阶段1 产出。每个子任务必须五要素齐全（目标/范围/输入输出/验收标准/依赖）。
> **并发组约定（方向④）**：按**文件落点不相交**把任务划入并发组（G1/G2/…）——**并发组恒为"落点白名单互不相交、可一条消息内机械并发的任务集"**（组内无写冲突）；两个任务落点相交（写同一文件）时**禁止同组并发**，裁决唯一见 `request-analysis` §4.1.2——**要么合并为同一任务**（由同一 generator 端到端独占该文件），**要么拆到不同并发组、在「依赖」标依赖序串行**。每个任务在五要素之外增一行「备注」承载**所属并发组 + 该组落点白名单（全限定路径）**；跨组依赖在「依赖」要素或下方总览表「并发性」列显式标依赖序。
