# 规则 · 项目编码规范

> 稳定不变约束。告诉 Agent「编码层面的标准是什么」。业务领域知识归 Wiki，不在此展开。
> **定制说明（M4 落地）**：本文由模板原版 v1.1.0 **重组**为两层加载结构——① **通用层**（语言无关语义硬约束 · 跨栈恒等 · §2 GEN-1~7）② **栈特定层**（Python §3 PY-1~5 / React+Vite §4 FE-1~7）。既有 R-001/R-002/R-005 + CODE-001~007 **语义不变**：语义本质归通用层、框架载体归栈特定层，二者合起等价于既有语义 + FE 对等扩展（映射见 §5 · 无双源矛盾定义）。承接 S-004（编码规范分通用/栈特定两层）/ ADR-003（流程层 · profile 拆分）。
> **（溯源 · RM-2026-131 / S-004）**：上述通用层（§2 GEN）/ 栈特定层（§3 PY · §4 FE）两层重组即 roadmap 卡 **RM-2026-131**（scope S-004）的交付内容，由元流程 `proj-module-fullstack-plugin-20260627` M4 工程基线 commit `b275906` 落地，经变更卡 `chore-rm131-s004-formalize-20260628` 独立审计与形式化闭环。

## 0. 硬约束登记（MUST 机械，供 docs/stage-01-Harness体系建设/03-质量与改进/08-质量门禁与反馈回路规范.md 转校验项）

> 本节是机器校验**登记索引**（非规则定义体）——R-编号供门禁规范转校验项消费，编号与语义稳定保留。**语义本体在 §2 通用层 GEN-x**（跨栈映射载体在 §3/§4），本节只登记机械校验入口、不重复定义（单源 · C-3）。

| 编号 | 约束 | 通用层语义 | 校验载体 |
|---|---|---|---|
| R-001 | 金额/价格字段用 **整数最小单位（分）**，禁止用 `float`/`double` 存储金额 | GEN-1 | ruff / mypy / lint / 评审 |
| R-002 | 所有外部 HTTP/RPC 调用必须设置**超时**与**降级** | GEN-2 | 静态扫描 + 评审 |
| R-005 | 禁止将密钥/令牌/私钥提交入库 | GEN-3 | secret 扫描 hook / CI |

> **R-002 评审强绑定**：R-002 / GEN-2 / CODE-002 / PY-3 / FE-3 违规 = `code-review` 评审报告**默认 MUST FIX 档**；豁免须在评审报告中**显式声明**为 `[HITL-3 豁免 · YYYY-MM-DD · <理由>]` 并经人工暂停确认（HITL-3）。

## 1. 适用范围 + 加载模型

适用于本项目所有源码的编码写法约束（全栈 · Python 后端 + React+Vite 前端）。

- **通用层（§2）**：全程加载（任何栈适用 · 语言无关语义）。
- **栈特定层（§3 Python / §4 React+Vite）**：按 `HARNESS_CONFIG.yaml` 的 `stack` 加载对应 active profile 一层（python ∨ react-vite）；新增栈 = 加一层栈特定层，通用层不变（对应 ADR-003 AR-LY-1 流程层↔绑定层单向依赖）。
- **业务仓覆盖**：复制 Harness 到其他语言/框架时新增对应栈特定层，但不得与通用层 GEN-x 语义冲突。

## 2. 通用层（GEN · 语言无关语义 · 跨栈恒等）

> 这些是**语义**约束，不绑定具体语言/框架；既有 R-001/R-002/R-005 + CODE-001~007 的语义本质归此层（框架载体在栈特定层 §3/§4）。

| 编号 | 规则（语义） | 级别 | 跨栈映射 |
|---|---|---|---|
| GEN-1 | **金额/价格用整数最小单位（分），禁浮点存金额**；折扣等用整数运算（= R-001/CODE-001 语义） | MUST 机械 | Python `price_cents: int` / TS `priceCents: number`（整数语义 · 禁 float 金额） |
| GEN-2 | **外部 HTTP/RPC 调用必须设超时 + 降级**（= R-002/CODE-002 语义） | MUST 机械+人工 | Python httpx `timeout=` / TS fetch+AbortController 或 axios `timeout` + 失败降级 |
| GEN-3 | **禁密钥/令牌入库**，敏感配置走环境变量（= R-005/CODE-005 语义 · 含厂商层引擎 API key 类目 · 守 C-1 不点名） | MUST 机械 | 跨栈 secret 扫描 |
| GEN-4 | **表现/编排层只做校验+编排+委托，业务逻辑下沉**（= CODE-003 语义） | MUST 人工 | Python router 瘦身→service / FE components 瘦身→hooks/containers（ES-FE-3） |
| GEN-5 | **结构化日志，禁裸输出打生产路径**（= CODE-004 语义） | SHOULD | Python `logging`/structlog 禁 print / TS logger 禁裸 console 打生产 |
| GEN-6 | **异常分层不吞底层异常，禁裸 catch 不处理**（= CODE-006 语义） | SHOULD | Python 禁裸 `except:` / TS 禁空 `catch {}` 吞错 |
| GEN-7 | **优先复用项目内共享模块，禁重复造轮子**（= CODE-007 语义 · 防熵增） | SHOULD | 跨栈评审 |

## 3. 栈特定层 · Python（FastAPI / pytest / ruff）

> 既有 `项目编码规范.md` 的 Python/FastAPI 框架载体归此层（语义在通用层 §2）。

| 编号 | 规则 | 级别 | 校验 |
|---|---|---|---|
| PY-1 | API 层用 FastAPI；业务逻辑在 `service` 包；router 不写大段逻辑（GEN-4 的 Python 载体 = CODE-003） | MUST 人工 | ruff + 评审 |
| PY-2 | 金额 `int`（分）+ 整数折扣运算（GEN-1 Python 载体 = CODE-001/R-001） | MUST 机械 | ruff/mypy + 评审 |
| PY-3 | httpx 调用设 `timeout=` + 失败降级（GEN-2 Python 载体 = CODE-002/R-002） | MUST 机械+人工 | 静态扫描 + 评审 |
| PY-4 | 测试用 pytest；测试放 `tests/`；门禁等价命令 `pytest -q`（解析器 → `{exit,total,passed}` · S-001） | MUST 机械 | pytest + 解析器单测 |
| PY-5 | 包/模块/文件 `snake_case`；Pydantic 模型放 `model`/`schemas`（= ES-004） | SHOULD | ruff + 评审 |

## 4. 栈特定层 · React + Vite + TypeScript（vitest）

> 首个 FE profile（D-前端 · S-005/S-006）；与 Python 层对等。

| 编号 | 规则 | 级别 | 校验 |
|---|---|---|---|
| FE-1 | TypeScript strict（`tsconfig` strict: true）；禁 `any` 滥用（显式类型边界） | SHOULD | tsc + 评审 |
| FE-2 | 金额用整数分（`priceCents: number` 整数语义），禁浮点存金额（GEN-1 FE 载体）；金额计算整数运算 | MUST 机械+人工 | lint + 评审 |
| FE-3 | services 层外部调用设超时（AbortController/axios timeout）+ 失败降级（GEN-2 FE 载体 = ES-FE-5） | MUST 人工 | 评审 |
| FE-4 | 组件分层遵 `工程结构.md` ES-FE-1~5（components 纯展示 / 逻辑在 hooks/containers / 外部调用在 services）（GEN-4 FE 载体） | MUST 机械+人工 | import 方向检查 + 评审 |
| FE-5 | 测试用 vitest；用例 `*.test.tsx`/`*.test.ts`；门禁等价命令 `vitest run`（解析器 → `{exit,total,passed}` · S-001 · 跨 profile GateContract 恒等） | MUST 机械 | vitest + 解析器单测 |
| FE-6 | 组件 `PascalCase` / hooks `useXxx` / services `xxxService`（= ES-FE-4）；结构化日志禁裸 console 打生产（GEN-5 FE 载体） | SHOULD | lint + 评审 |
| FE-7 | 前端 `model/` TS 类型与后端 schema 对齐（金额单位/字段名等通用约束跨栈一致 · 联调契约 S-007） | MUST 人工 | 评审 + fe-integration 收敛 |

## 6. 正例 / 反例

- 正例：BE `price_cents: int = 9999` 表金额（分 · PY-2/GEN-1）；FE `priceCents: number` 表金额分（FE-2/GEN-1）——同一通用语义两栈载体。
- 正例：FE example vitest 输出经解析器套 GateContract 判定，与 Python pytest 同判定式（FE-5 / PY-4 / S-001）。
- 反例：BE `price: float = 99.99` 表金额（违反 PY-2/GEN-1/R-001）；FE `price: number = 99.99` 浮点存金额（违 FE-2/GEN-1）。
- 反例：`httpx.get(url)` 未设 `timeout`（违 PY-3/GEN-2/R-002）；FE service `fetch(url)` 未设超时（违 FE-3/GEN-2）。
- 反例：在 FastAPI 路由函数内写批量询价 + 折扣计算，未委托 `service`（违 PY-1/GEN-4/CODE-003）。
- 反例：把 FastAPI 专属约定写进通用层 GEN 段（违 §1 加载模型——框架载体应在栈特定层 PY 段）。

## 7. 例外与豁免

- 偏离硬约束（GEN / R- / MUST 类）原则上不允许；确有必要须经 Reviewer + 人工 HITL 批准，并在 `summary.md` 记录理由、影响面与后续整改计划。
