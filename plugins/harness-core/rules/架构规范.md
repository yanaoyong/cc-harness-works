# 规则 · 架构规范（三 plugin 边界 + 分层依赖 + 有状态/无状态边界）

> 基于 C4 容器图（c4-container.md）+ ADR-001 平移矩阵 + ADR-003 三 plugin 拆分。本规范管「plugin 边界 / 分层依赖方向 / 有状态-无状态边界 / 旁路组件边界 / 厂商隔离边界」五类架构约束；既有 `工程结构.md` ES-001~007 管业务分层目录树，本规范**增补不替代**。
> **落地说明（M4）**：本文由 M3.3 架构规范设计稿物理落地为正式 Rules（C-3 单源 · 设计记录保留于 m3.3_customization/rules/ DRAFT）。本规范为**按需加载（L2）**架构判断规则，非 L1 常驻。

## 0. 适用范围

适用于 Harness 模板仓自身的架构组织（三 plugin 拆分 + 流程层/绑定层依赖 + 有状态/无状态制品落点 + 旁路组件）。业务/Demo 代码分层依赖见 `工程结构.md` ES-001~006 / ES-FE-1~5。

## 1. 三 plugin 边界（AR-PLUGIN · ADR-003）

| 编号 | 规则 | 级别 | 校验 |
|---|---|---|---|
| AR-PL-1 | 三 plugin 边界固定：`harness-core`（栈无关方法论骨架 + 双流程 + 旁路组件）/ `harness-profile-python` / `harness-profile-react-vite`；同 marketplace 并列 | MUST 人工 | 评审（manifest 边界核对）|
| AR-PL-2 | **栈特定资产不得进核心 plugin**：pytest/vitest/ruff/FastAPI/React 等栈细节只能在对应 profile plugin；核心 plugin 流程层零栈特定串 | MUST 机械 | grep 核心 plugin 流程层基线为空（S-002）|
| AR-PL-3 | 双流程（10 阶段 K + 元流程 M0–M5）随核心 plugin **单包**（OQ-04 · 不拆包） | MUST 人工 | 评审 |
| AR-PL-4 | 新增技术栈 = 新增一份 profile plugin（声明等价命令 + 解析器 + 分层映射），**核心 plugin 0 改动** | MUST 机械 | 加 fixture profile，git diff 核心流程层为空（NFR-扩展-1/4）|

## 2. 分层依赖方向（AR-LAYER · ADR-003 单向依赖）

> （溯源 · RM-2026-127 / S-002a：流程层 6 skill 去栈特定串使 BindingLayerContract 抽象签名成为流程层唯一依赖面，坐实 AR-LY-1 流程层↔绑定层单向 · 契约定义权威源仍在 `proj-module-fullstack-plugin-20260627/m3_architecture/m3.2_interface/api-design.md §1.1`，此处仅追加溯源锚注、不动判定语义 C-3 单源）

| 编号 | 规则 | 级别 | 校验 |
|---|---|---|---|
| AR-LY-1 | **流程层（栈无关）↔ 绑定层（栈特定）单向**：流程层只依赖绑定层**抽象签名**（BindingLayerContract），不依赖任一具体 profile；profile 反向注册进绑定层、单向依赖核心 | MUST 机械 | grep + 依赖方向检查（BL-INV-2/3）|
| AR-LY-2 | 业务/Demo 代码沿用 ES-001/002（后端）/ ES-FE-1/2（前端）：单向向下，禁逆向依赖（工程结构.md 权威） | MUST 机械 | import 方向检查 |
| AR-LY-3 | 门禁判定式（GateContract）跨所有 profile 恒等，由流程层定义、profile 不得改判定本质（C-5） | MUST 机械 | 双向校验（API-BL-2）|

## 3. 有状态 / 无状态边界（AR-STATE · ADR-001 平移矩阵）

| 编号 | 规则 | 级别 | 校验 |
|---|---|---|---|
| AR-ST-1 | **无状态声明式资产**（skills/agents/hooks/commands/components bin/绑定层/manifest · ADR-001 M-1~M-8）平移进 plugin（装只读 cache） | MUST 人工 | 评审对照 ADR-001 矩阵 |
| AR-ST-2 | **有状态磁盘制品**（项目 CLAUDE.md / .harness/changes/ SSOT / HARNESS_CONFIG.yaml / wiki/ / .codegraph/ / _TEMPLATE 落盘 · ADR-001 R-1~R-6）**必须留项目侧**，plugin 不得代管（C-7） | MUST 人工 | 评审：plugin 资产无写项目目录假设（PM-INV-3）|
| AR-ST-3 | 跨张力桥 = SessionStart hook（无状态声明 → 每 session 投射到有状态项目侧 · ADR-002）；不得发明 PostInstall 等不存在的桥（CL-04） | MUST 人工 | 评审（与 ADR-002 一致）|
| AR-ST-4 | 会话宪法本体必留项目 CLAUDE.md（C-4 · CL-01）；plugin 侧只重建可重建子集，不取代 | MUST 人工 | 评审 + 行为验收（RC-INV-1/2）|

## 4. 旁路组件边界（AR-BYPASS · ADR-005 既有项目级 · C-2）

| 编号 | 规则 | 级别 | 校验 |
|---|---|---|---|
| AR-BY-1 | 旁路组件（codegraph / wiki-engine / research-discovery）为查询/证据工具，**不进任一阶段门禁判定式、不夺裁决权**（ADR-005 · C-2） | MUST 人工 | 评审：门禁判定式无旁路组件引用 |
| AR-BY-2 | 组件本体（bin+skill）无状态、平移进核心 plugin（ADR-001 M-5）；组件产物（wiki/ / .codegraph/）有状态、留项目侧（R-4/R-5）；`.codegraph/` 不入仓（ADR-004 · cg init 重建） | MUST 人工 | 评审 + 落点核对 |
| AR-BY-3 | 外部可执行组件落 `.harness/components/<comp>/`（ES-007），符号链接 `.claude/skills/<comp>` 注册；组件路径须相对化可移植（S-010 · 消硬编码） | MUST 人工 | 评审 + 非默认深度安装不报路径错 |

## 5. C-1 厂商隔离边界（AR-VENDOR · 铁律）

| 编号 | 规则 | 级别 | 校验 |
|---|---|---|---|
| AR-VN-1 | `.harness/` 抽象契约层**零具体外部检索引擎/后端硬引用**；research-discovery 仅以抽象 skill 名出现，引擎绑定在厂商层 | MUST 机械 | grep `.harness/` 抽象层无引擎专名 |

## 6. 正例 / 反例

- 正例：新增 Vue profile plugin，核心 plugin 流程层 git diff 为空（AR-PL-4 / AR-LY-1）。
- 正例：`_TEMPLATE` 落盘走 SessionStart hook diff 探测，不假设 PostInstall（AR-ST-3）。
- 反例：把 `pytest` 解析逻辑写进核心 plugin 的 unit-test-ci 流程层 skill（违 AR-PL-2 / AR-LY-1）。
- 反例：让 codegraph 影响面查询结果进入阶段4门禁判定式当裁决依据（违 AR-BY-1 旁路语义）。
- 反例：试图让 plugin 安装时写项目 `.harness/changes/`（违 AR-ST-2 / C-7——plugin 只读 cache 写不了项目）。
